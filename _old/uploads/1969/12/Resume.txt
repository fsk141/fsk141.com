			<h2 class="entry-title"><a href="http://fsk141.com/?p=54" title="Permalink to SICP, LISP, Clisp, GCL, Guile, LISP is kewl&#8230;" rel="bookmark">SICP, LISP, Clisp, GCL, Guile, LISP is kewl&#8230;</a></h2>

			<div class="entry-meta">
				<span class="meta-prep meta-prep-author">Posted on</span> <a href="http://fsk141.com/?p=54" title="12:00 am" rel="bookmark"><span class="entry-date">November 24, 2008</span></a> <span class="meta-sep">by</span> <span class="author vcard"><a class="url fn n" href="http://fsk141.com/?author=1" title="View all posts by fsk141">fsk141</a></span>			</div><!-- .entry-meta -->

				<div class="entry-content">
				<p>So I&#8217;ve gone back to the stoneage, and am learning <a href="http://en.wikipedia.org/wiki/Lisp_(programming_language)">LISP</a> (circa 1958). I&#8217;m doing this not only to get a grasp on programming, but to learn the logic of programming. I&#8217;m going by the book and am using <a href="http://mitpress.mit.edu/sicp">Structure and Interpretation of Computer Programs</a> (SICP). It&#8217;s avaliable for free online, and is an awesome book. This sums up the book (it&#8217;s the first page):</p>
<p>This book is dedicated, in respect and admiration, to the spirit that lives in the computer.</p>
<blockquote><p>I think that it&#8217;s extraordinarily important that we in computer science keep fun in computing. When it started out, it was an awful lot of fun. Of course, the paying customers got shafted every now and then, and after a while we began to take their complaints seriously. We began to feel as if we really were responsible for the successful, error-free perfect use of these machines. I don&#8217;t think we are. I think we&#8217;re<br />
responsible for stretching them, setting them off in new directions, and keeping fun in the house. I hope the field of computer science never loses its sense of fun. Above all, I hope we don&#8217;t become missionaries. Don&#8217;t feel as if you&#8217;re Bible salesmen. The world has too many of those already. What you know about computing other people will learn. Don&#8217;t feel as if the key to successful computing is only in your hands. What&#8217;s in your hands, I think and hope, is intelligence: the ability to see the machine as more than when you were first led up to it, that you can make it more.</p></blockquote>
<p><i>Alan J. Perlis (April 1, 1922-February 7, 1990)</i></p>
<p>Onto the environment. I&#8217;ve found three LISP environments.</p>
<p><a href="http://clisp.cons.org">Clisp</a> (General LISP)</p>
<p><a href="http://www.gnu.org/software/gcl">GCL</a> (General LISP)</p>
<p><a href="http://www.gnu.org/software/guile/guile.html">Guile</a> (Scheme)</p>
<p>Since the book deals with scheme as opposed to general lisp, I&#8217;m probably going to use guile. I&#8217;ve just been introduced to the world of lisp, so my pick might change as time goes on. </p>
							</div><!-- .entry-content -->

			<h2 class="entry-title"><a href="http://fsk141.com/?p=53" title="Permalink to OpenSolaris | Synergy" rel="bookmark">OpenSolaris | Synergy</a></h2>

			<div class="entry-meta">
				<span class="meta-prep meta-prep-author">Posted on</span> <a href="http://fsk141.com/?p=53" title="12:00 am" rel="bookmark"><span class="entry-date">November 24, 2008</span></a> <span class="meta-sep">by</span> <span class="author vcard"><a class="url fn n" href="http://fsk141.com/?author=1" title="View all posts by fsk141">fsk141</a></span>			</div><!-- .entry-meta -->

				<div class="entry-content">
				<p>Recently I&#8217;ve started using OpenSolaris 2008.11 for work and play. It&#8217;s kinda a pain to get used to cause I&#8217;m so used to the linux way of doing things.</p>
<p>Such as: <br />
     Basic list all with ps:</p>
<pre class="code">
<b>Linux</b>  |  <b>OpenSolaris</b>
---------------------
ps ax  |  ps -Af
</pre>
<p>     Services in OpenSolaris:</p>
<pre class="code">
<b>Linux</b>     |  <b>OpenSolaris</b>
------------------------
service   |  svcs -a (list all services running)
init.d    |  svcadm (to start, stop, and restart)
rc.d      |
</pre>
<p>&#8212;&#8212;</p>
<p>Since linux has been my primary OS for so long, and I love <a href="http://www.archlinux.org>Arch Linux</a> so much, instead of ditching linux all the way. I&#8217;m doing a half switch to OpenSolaris. To achieve this I&#8217;m using <a href="http://synergy2.sourceforge.net</a>. It&#8217;s really easy to setup. I compiled it both on the OpenSolaris box, and my Arch Linux box, and put the following config on both of the machines:</p>
<p>/etc/synergy.conf</p>
<pre class="code">
section: screens
        Osol:
        Macarch:
end

section: links
        Osol:
                left = Macarch
        Macarch:
                right = Osol
end
</pre>
<p>Then all I have to do is run &#8216;synergys&#8217; on the host computer (the one with the keyboard &#038; mouse), and synergyc 10.200.2.10 (the ip address of the host)</p>

			<h2 class="entry-title"><a href="http://fsk141.com/?p=55" title="Permalink to Neato CSS code block" rel="bookmark">Neato CSS code block</a></h2>

			<div class="entry-meta">
				<span class="meta-prep meta-prep-author">Posted on</span> <a href="http://fsk141.com/?p=55" title="12:00 am" rel="bookmark"><span class="entry-date">November 23, 2008</span></a> <span class="meta-sep">by</span> <span class="author vcard"><a class="url fn n" href="http://fsk141.com/?author=1" title="View all posts by fsk141">fsk141</a></span>			</div><!-- .entry-meta -->

				<div class="entry-content">
				<pre class="code">
I'm a code block hooray!
</pre>
<p>How do I do it?<br />
Start by putting this in your css template:</p>
<pre class="code">
/* Code Block */
pre, .code {
        padding: 10px 15px;
        margin: 5px 0 15px;
        border-left: 5px solid #666666;
        background: #333333;
        font: 1em/1.5 "Courier News", monospace;
        color: #bbbbbb;
        overflow : auto;
}
</pre>
<p>Then to activate the code block do the following:</p>
<pre class="code">
&lt;pre class="code">
--code--
&lt;/pre>
</pre>
